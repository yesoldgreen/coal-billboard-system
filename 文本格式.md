# 文本格式

本文档用于说明当前项目后端已支持的文本格式定义，方便查看每种文本格式对应的业务含义、固定参数和接口参数命名。

当前版本实际支持的质量/价格模块文本格式只有：

- `2001`
- `2002`

---

## 文本格式 `2001`

### 基本信息

- 文本格式编号：`2001`
- 适用模块中文名：质量/价格
- 适用模块英文名：`quality`
- 煤源名称中文名：汇能长滩煤矿
- 煤源名称英文参数名：`coal_source_name`
- 内容中文名：价格
- 内容英文参数名：`content_label`

### 固定参数对照

| 中文名 | 英文名 | 值 |
| --- | --- | --- |
| 文本格式编号 | `format_code` | `2001` |
| 模块类型 | `module_type` | `quality` |
| 煤源名称 | `coal_source_name` | `汇能长滩煤矿` |
| 内容 | `content_label` | `价格` |

### 可识别内容

- 价格执行时间
- 煤种
- 含税价格
- 较上期变动

### 主要字段中英文对照

| 中文名 | 英文名 |
| --- | --- |
| 原始文本 | `raw_text` |
| 文本格式编号 | `format_code` |
| 模块类型 | `module_type` |
| 煤源名称 | `coal_source_name` |
| 内容 | `content_label` |
| 价格执行时间 | `priceExecutionTime` |
| 数据项列表 | `items` |
| 数据源名称 | `source_name` |
| 原始煤种名称 | `raw_name` |
| 含税价格 | `price` |
| 较上期变动 | `change_value` |

### 文本特征

- 适用于“价格执行如下”这一类通知文本
- 支持在正文中识别执行时间
- 支持识别 `↑` / `↓` 变动值
- 如果某煤种未写涨跌，系统按当前规则写入 `0`

### 示例

```text
@所有人 经总经理办公会议研究决定，汇能长滩露天煤矿从2026年3月18日19点起(以空车过磅时间为准)价格执行如下：
1、汇能6号原煤执行455元/吨。（↑5元/吨）
2、汇能8号原煤执行345元/吨。（↑5元/吨）
```

## 文本格式 `2002`

### 基本信息

- 文本格式编号：`2002`
- 适用模块中文名：质量/价格
- 适用模块英文名：`quality`
- 煤源名称中文名：白家海子煤矿
- 煤源名称英文参数名：`coal_source_name`
- 内容中文名：价格
- 内容英文参数名：`content_label`

### 固定参数对照

| 中文名 | 英文名 | 值 |
| --- | --- | --- |
| 文本格式编号 | `format_code` | `2002` |
| 模块类型 | `module_type` | `quality` |
| 煤源名称 | `coal_source_name` | `白家海子煤矿` |
| 内容 | `content_label` | `价格` |

### 可识别内容

- 价格执行时间
- 煤种
- 含税价格
- 较上期变动

### 主要字段中英文对照

| 中文名 | 英文名 |
| --- | --- |
| 原始文本 | `raw_text` |
| 文本格式编号 | `format_code` |
| 模块类型 | `module_type` |
| 煤源名称 | `coal_source_name` |
| 内容 | `content_label` |
| 价格执行时间 | `priceExecutionTime` |
| 数据项列表 | `items` |
| 数据源名称 | `source_name` |
| 原始煤种名称 | `raw_name` |
| 含税价格 | `price` |
| 较上期变动 | `change_value` |

### 文本特征

- 当前解析规则与 `2001` 相同
- 只是业务上绑定的煤源名称不同
- 返回更新备注时会生成：`白家海子煤矿最新价格信息`

### 示例

```text
调价通知:
所有客户，联海煤业白家海子煤矿自2026年3月19日12时00分起（以空车回皮为准），各煤种含税价格调整如下:
1、汇能1号洗沫煤执行431元/吨（↑21元/吨）；
```
