# v1.7.1 微信机器人接口文档

本文档提供给微信机器人调用方。当前接口用于把原始文本数据直接提交到告示牌服务端，由服务端自动解析并写入指定告示牌模块。

## 概览

- 协议：HTTP/HTTPS
- 方法：`POST`
- 路径：`/api/inbound/autofill`
- 返回：`application/json; charset=utf-8`

建议正式环境统一通过 HTTPS 暴露接口。

## 重要部署注意点

这部分请务必同步给部署同事和微信机器人开发同事。

1. 正式环境必须走 HTTPS

- 建议通过 Nginx 或云负载均衡统一接入 HTTPS
- 不建议让机器人直接请求裸露的 HTTP 端口

2. 反向代理必须透传请求头

至少要确保以下请求头不会被吃掉或改写：

- `X-Billboard-Id`
- `X-Module-Type`
- `X-Format-Code`
- `Content-Type`

3. 建议固定使用 UTF-8

- `application/json; charset=utf-8`
  或
- `text/plain; charset=utf-8`

4. 机器人侧不要自动重试 4xx

- `4xx` 一般代表参数、格式、映射错误
- `5xx` 才适合做有限重试

## 请求头

以下请求头建议全部显式传递：

- `X-Billboard-Id: 1`
- `X-Module-Type: quality`
- `X-Format-Code: 2001`

说明：

- `X-Billboard-Id`
  目标告示牌 ID，必须是整数。
- `X-Module-Type`
  目标模块。
  当前仅支持：
  - `queue`
  - `quality`
- `X-Format-Code`
  文本格式编号。
  当前支持：
  - `queue` 模块：使用现有排队拉运格式文本
  - `quality` 模块：`2001`、`2002`

## 请求体格式

接口兼容两种请求体格式。

### 方式一：`application/json`

推荐用于大多数机器人服务。

```http
Content-Type: application/json
```

请求体：

```json
{
  "raw_text": "@所有人 经总经理办公会议研究决定，汇能长滩露天煤矿从2026年3月18日19点起(以空车过磅时间为准)价格执行如下：1、汇能6号原煤执行455元/吨。（↑5元/吨）"
}
```

### 方式二：`text/plain`

适合直接转发原始文本。

```http
Content-Type: text/plain
```

请求体示例：

```text
@所有人 经总经理办公会议研究决定，汇能长滩露天煤矿从2026年3月18日19点起(以空车过磅时间为准)价格执行如下：
1、汇能6号原煤执行455元/吨。（↑5元/吨）
2、汇能8号原煤执行345元/吨。（↑5元/吨）
```

## 可选 Body 透传字段

如果调用方不方便设置自定义请求头，也可以把目标信息放进 JSON body 中。服务端会优先读取请求头，缺失时再读取 body。

```json
{
  "billboard_id": 1,
  "module_type": "quality",
  "format_code": "2001",
  "raw_text": "..."
}
```

仍然建议正式环境优先使用请求头，因为这更符合网关、Nginx、日志排查和跨服务转发场景。

## 成功响应

```json
{
  "success": true,
  "status": "applied",
  "log_id": 12,
  "billboard_id": 1,
  "billboard_name": "汇能长滩 煤矿内参",
  "module_type": "quality",
  "format_code": "2001",
  "updated_count": 4,
  "unmapped_sources": [],
  "missing_rows": [],
  "price_execution_time": "2026年3月18日19点起(以空车过磅时间为准)"
}
```

字段说明：

- `updated_count`
  本次成功更新的已映射煤种数量。
- `unmapped_sources`
  本次识别到了，但系统中尚未建立映射的煤种。
- `missing_rows`
  已有映射，但目标告示牌表单中没有对应行的煤种。
- `price_execution_time`
  本次识别出的价格执行时间。若文本未包含，则为 `null`。

## 失败响应

### 1. 目标参数错误

```json
{
  "success": false,
  "status": "invalid_target",
  "error": "billboard_id 无效"
}
```

### 2. 模块不支持

```json
{
  "success": false,
  "status": "unsupported_module",
  "error": "module_type 仅支持 queue 或 quality"
}
```

### 3. 缺少格式编号

```json
{
  "success": false,
  "status": "missing_format_code",
  "error": "缺少 format_code"
}
```

### 4. 正文为空

```json
{
  "success": false,
  "status": "empty_payload",
  "error": "raw_text 不能为空"
}
```

### 5. 目标告示牌不存在

```json
{
  "success": false,
  "status": "billboard_not_found",
  "error": "目标告示牌不存在"
}
```

### 6. 格式不支持

```json
{
  "success": false,
  "status": "unsupported_format",
  "log_id": 18,
  "error": "当前不支持质量/价格模块格式 2003"
}
```

### 7. 解析或入库失败

```json
{
  "success": false,
  "status": "parse_or_apply_failed",
  "log_id": 19,
  "error": "未识别到有效的质量/价格数据"
}
```

## 当前支持格式

### `quality + 2001`

支持解析：

- 价格执行时间
- 煤种
- 含税价格
- 较上期变动

规则：

- `↑5` 自动转为 `+5`
- `↓5` 自动转为 `-5`
- 若文本中该煤种未写涨跌，则自动写入 `0`
- 热值、灰分、硫等未在文本中出现的字段保持原值

### `quality + 2002`

规则与 `2001` 相同。

### `queue`

当前支持现有排队拉运自动填充格式：

```text
汇能2号洗混块煤: 103
汇能1号洗混中块煤: 675
```

## 部署与兼容性建议

适用于阿里云、腾讯云、物理机或容器环境。

### 1. Nginx 反向代理

请确保反向代理不会丢失以下请求头：

- `X-Billboard-Id`
- `X-Module-Type`
- `X-Format-Code`
- `Content-Type`

### 2. 编码

建议统一使用：

- `UTF-8`
- `application/json; charset=utf-8`
  或
- `text/plain; charset=utf-8`

### 3. 超时与重试

建议微信机器人侧：

- 请求超时设置为 10 秒
- 对 5xx 错误可有限重试
- 对 4xx 错误不要自动重试，应记录并人工排查

## 注意事项

- 接口不会自动创建新映射关系。
- 若识别到的数据源名称尚未配置映射，会在响应的 `unmapped_sources` 中返回。
- 若已存在映射，但目标表单没有对应煤种行，会在 `missing_rows` 中返回。
- 当前接口不支持广告自动写入。
- 当前接口不支持物流费用模块。
