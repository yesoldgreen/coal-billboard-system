# 快速入门指南

## 系统概述

煤矿内参告示牌系统是一个完整的数据展示解决方案，包含：
- **后台管理系统**: 用于录入和管理告示牌数据
- **客户端展示系统**: 用于向用户展示实时数据

## 5分钟快速体验

### 步骤1: 安装依赖（约2分钟）

```bash
cd coal-billboard-system
./install.sh
```

如果没有安装脚本执行权限，可以手动执行：
```bash
chmod +x install.sh
./install.sh
```

### 步骤2: 启动所有服务（约1分钟）

```bash
./start-all.sh
```

服务启动后，你会看到：
- 后端服务: http://localhost:3000
- 后台管理: http://localhost:3001
- 客户端: http://localhost:3002

### 步骤3: 登录后台管理（30秒）

1. 打开浏览器访问 http://localhost:3001
2. 输入登录信息：
   - 用户名: `hulianshikong`
   - 密码: `hlsk2026`
3. 点击"登录"

### 步骤4: 创建第一个告示牌（1分钟）

1. 点击"创建告示牌"按钮
2. 输入告示牌名称，例如：`汇能长滩 煤矿内参`
3. 点击"确定"

### 步骤5: 编辑告示牌数据（2分钟）

1. 在告示牌列表中，点击刚创建的告示牌的"编辑数据"按钮
2. 编辑"排队拉运"数据：
   - 点击"添加行"
   - 填入坑口名称（如：汇能2号）
   - 填入已磨约、排队中、已叫号、已入场的数据
   - 可以添加多行数据
   - 可选：添加广告信息
   - 点击"保存所有数据"

3. 切换到"质量/价格"标签页：
   - 点击"添加行"
   - 填入坑口、热值、灰分、硫、含税价格、变动等数据
   - 可选：添加广告信息
   - 点击"保存所有数据"

4. 切换到"物流费用"标签页：
   - 点击"添加行"
   - 选择类型（直运或集运站）
   - 填入相应的物流费用信息
   - 可选：添加广告信息
   - 点击"保存所有数据"

### 步骤6: 查看客户端展示（30秒）

1. 回到告示牌列表页面
2. 点击"复制链接"按钮
3. 在新标签页中粘贴并访问该链接
4. 查看告示牌的展示效果

🎉 恭喜！你已经成功创建并运行了第一个告示牌！

## 完整操作流程

### 管理告示牌

#### 创建告示牌
1. 登录后台管理
2. 点击"创建告示牌"
3. 输入告示牌名称
4. 点击"确定"

#### 修改告示牌名称
1. 在列表中找到要修改的告示牌
2. 点击"改名"按钮
3. 修改名称
4. 点击"确定"

#### 删除告示牌
1. 在列表中找到要删除的告示牌
2. 点击"删除"按钮
3. 确认删除

### 编辑数据

#### 排队拉运数据
- **字段说明**:
  - 坑口: 煤矿坑口名称，如"汇能2号"
  - 已磨约: 已签订磨运合约的数量
  - 排队中: 正在排队的车辆数
  - 已叫号: 已叫号但未入场的车辆数
  - 已入场: 已经入场的车辆数

- **操作**:
  - 点击"添加行"添加新数据行
  - 在表格中直接编辑数据
  - 点击"删除"移除某一行
  - 添加广告（可选）
  - 点击"保存所有数据"

#### 质量/价格数据
- **字段说明**:
  - 坑口: 煤矿坑口名称
  - 热值: 煤炭热值，单位通常是千卡/千克
  - 灰分: 煤炭中灰分含量百分比
  - 硫: 煤炭中硫含量百分比
  - 含税价格: 含税的煤炭价格
  - 较上期变动: 相比上期的价格变动，如"+10"或"-5"

- **操作**: 同排队拉运数据

#### 物流费用数据
- **类型说明**:
  - 直运: 直接运输，填写起点、终点和运费
  - 集运站: 经过集运站，需填写集运站名称、站运费、站台费

- **操作**: 同排队拉运数据

### 查看客户端

1. 在告示牌列表中点击"复制链接"
2. 链接格式：`http://localhost:3002/billboard/{告示牌ID}`
3. 访问链接即可查看告示牌
4. 数据会每30秒自动刷新
5. 可以分享链接给其他用户

## 数据更新机制

### 实时性
- 后台更新数据后，客户端在下次轮询（最多30秒）后会自动更新
- 每个模块显示最后更新时间

### 无历史数据
- 系统不保存历史数据
- 每次更新会覆盖之前的数据
- 如需历史记录，请手动备份

## 常见问题

### Q: 如何停止服务？
A: 执行 `./stop-all.sh` 或手动 kill 进程

### Q: 如何修改端口？
A: 编辑各服务目录下的 `.env` 文件

### Q: 数据存储在哪里？
A: SQLite 数据库文件位于 `backend/billboard.db`

### Q: 如何备份数据？
A: 复制 `backend/billboard.db` 文件

### Q: 客户端能否离线访问？
A: 不能，客户端需要连接后端API获取数据

### Q: 支持多少个告示牌？
A: 理论上无限制，取决于服务器性能

### Q: 如何修改登录密码？
A: 目前需要直接修改数据库或联系管理员

### Q: 底栏二维码如何添加？
A: 当前版本底栏是静态的，未来版本会支持后台上传

## 生产环境部署

### 安全建议
1. 修改默认密码
2. 使用 HTTPS
3. 配置防火墙
4. 定期备份数据库
5. 设置 CORS 白名单

### 性能优化
1. 使用 PM2 管理进程
2. 配置 Nginx 反向代理
3. 启用 gzip 压缩
4. 使用 CDN 加速静态资源

### 监控建议
1. 监控服务器资源使用
2. 监控 API 响应时间
3. 记录错误日志
4. 设置告警通知
